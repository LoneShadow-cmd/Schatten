## 4. VISUAL DIRECTION FOR EACH SECTION

---

### NAVIGATION
**Layout:** Sticky, full-width. Background: deep black with 8% opacity glass blur on scroll. Logo left, links center, CTA right.
**Imagery:** None. Purely typographic.
**Color:** Logo in pure white. Nav links in muted silver (#B0B0B0). CTA button: ghost, 1px white border, white text.
**Motion:** Nav appears with 400ms fade on page load. On scroll past 100px, dark glass background fades in (300ms). Nav links underline on hover with a 200ms left-to-right sweep.
**Emotional intent:** Authority and restraint. The navigation does not compete for attention — it serves navigation only.

---

### SECTION 01 — HERO
**Layout:** Full-viewport height. 3D katana floats at center-right (60% horizontal position), occupying 50–60% of viewport height. Text block lives in center-left third. Deep ambient glow behind the blade.
**3D / Art direction:**
- Katana rendered in hyper-realistic PBR materials: steel blade with fine hamon (temper line), lacquered black tsuka (handle), minimal gold tsuba (guard).
- Environment: near-total darkness with a single deep-blue/silver ambient light source from above-left, casting a reflection along the blade edge.
- Floating animation: slow sine-wave float, ±12px vertical, 6-second cycle.
- Rotation: slow Y-axis rotation, 0.3°/second on idle. On cursor movement, the blade subtly tracks cursor position (max 8° tilt in any direction) using parallax — not full rotation.
- Light interaction: as cursor moves, the reflected highlight on the blade shifts to catch the light dynamically.
- Depth of field: blade in sharp focus, background in cinematic blur.
- Particle layer: 6–8 extremely slow-moving motes of light near the blade tip — not flashy, barely visible, like dust in a still room.
**Color:** Background: #050508 (near-black with a cold blue undertone). Text: pure white. Eyebrow: #8A8FA0 (cool silver-gray). Blade glow: deep cerulean (#1E3A5F at 30% opacity).
**Texture:** Subtle noise texture overlay on background (2% opacity) to add depth and prevent digital flatness.
**Motion:** On page load, the hero fades in over 1.2 seconds. Headline words enter with a staggered reveal (each word slides up from 20px below, fades in, 80ms delay between words). CTA button enters last with an upward fade. Katana enters with a 600ms slow materialization (opacity 0 → 1 with a slight scale 0.96 → 1.0).
**Emotional intent:** Awe, stillness, power. The first 3 seconds should feel like walking into a very quiet, very serious room.

---

### SECTION 02 — PROBLEM
**Layout:** Single-column text block, max-width 800px, centered. Problem pillars below as a 2×3 grid of minimal bordered cards.
**Imagery:** No images. Typography-forward. Optional: extremely subtle abstract line texture in background suggesting complexity/noise — then the clean text becomes the visual relief.
**Color:** Background: #070709. Text: #E8E8EE (off-white, slightly cool). Problem pillar cards: 1px border in #1C1C28, background #0D0D15, text #A0A0B0.
**Motion:** On scroll-entry, body paragraphs animate with a staggered paragraph reveal — each fades in from 0 opacity with 15px upward translate, 120ms stagger between paragraphs. Problem pillars enter with a left-to-right cascade reveal, 100ms stagger.
**Emotional intent:** Recognition and discomfort. The visitor should feel described, not sold to. The animation pacing should be slightly slower here — deliberate, serious, measured.

### HERO ANIMATION SYSTEM

**Technology:** HTML5 Canvas API (no Three.js — eliminates abstraction overhead), GSAP for all UI/scroll animations, Lenis for smooth scroll. The katana animation is entirely canvas-driven via a webp frame sequence, controlled by scroll progress.

**Katana Canvas Behavior (Implemented — see Section 6):**
```
Scroll-Driven State Machine:
- 362 webp frames extracted at 30fps from katana-animation.mp4
- frameIndex = floor(lerpedScrollProgress × totalFrames)
- Lerp factor: 0.06 — ~200ms of perceived inertia, weighted, not broken
- Adaptive lerp: 2.5× factor when scroll jump > 10% (keyboard/Page Down)

Layer Position:
- position: fixed; z-index: 2; mix-blend-mode: screen; pointer-events: none
- Layer 1: WebGL shader (atmospheric rings)
- Layer 2: Katana canvas (this system)
- Layer 3+: All copy and content

Micro-Parallax:
- Vertical offset = scrollVelocity × direction × −800, clamped ±8px
- Barely perceptible — reinforces depth, never distracting

Mobile:
- Every 2nd frame loaded (181 frames) — preserves perception, halves bandwidth
- Same canvas system, same scroll-driven behavior
```

## 6. ANIMATION SYSTEM — KATANA FLOW ENGINE

---

### Philosophy

The katana is not an object. It is the **origin point of a reactive system**.

All elements — camera, light, environment — behave as dependent variables responding to the katana's presence and movement. Nothing leads. The katana defines everything.

The experience is not animation-driven. It is **state-driven by scroll**, where each frame represents a precise moment within a continuous, uninterrupted motion field. The katana must feel as if it exists in a controlled physical environment governed by invisible forces: inertia, resistance, tension, and intention.

Every visual decision reinforces the perception that nothing is random.

---

### Stack

| Layer | Technology | Reason |
|-------|-----------|--------|
| Frame rendering | HTML5 Canvas API | Direct pixel-level authority — no abstraction overhead |
| Motion source | webp frame sequence (362 frames @ 30fps) | Absolute control over motion fidelity |
| Scroll driver | `requestAnimationFrame` + lerp | Syncs to browser paint cycle — no `setInterval` drift |
| UI/scroll animations | GSAP + ScrollTrigger | All non-katana animations |
| Smooth scroll | Lenis | Consistent scroll velocity across devices |

No Three.js. No GSAP for the katana. No timeline-based artificial motion.

---

### Canvas Architecture

```
position: fixed          — persists as layer 2 across entire page
top: 0; left: 0          — anchored to viewport origin
width: 100vw; height: 100vh
z-index: 2               — above shader (z-1), below all content (z-3+)
mix-blend-mode: screen   — dark frame backgrounds become transparent;
                           only luminous katana elements are visible
pointer-events: none     — all interactions pass through to content
opacity: 0.85            — precise visual weight, not overwhelming
```

**Cover-fit rendering:** Frame is scaled to fill the viewport while preserving aspect ratio. Centered horizontally and vertically with micro-parallax offset applied vertically.

**devicePixelRatio:** Canvas physical pixels = CSS pixels × min(dpr, 2). Sharp on all Retina/HiDPI displays.

---

### Scroll-to-Frame Mapping

```
rawProgress    = window.scrollY / (scrollHeight − viewportHeight)  [0→1]
smoothProgress = lerp(smoothProgress, rawProgress, factor)
frameIndex     = floor(smoothProgress × (totalFrames − 1))

Lerp factor:
  Normal scroll:  0.06  (~200ms perceived inertia — weighted, not broken)
  Large jump:     0.15  (2.5× — catches up for keyboard/Page Down jumps)
```

The lerp lives in `requestAnimationFrame`, never in the scroll event handler. Scroll events fire off the main thread — lerp must sync to the paint cycle for true 60fps smoothness.

---

### Motion Rules (Non-Negotiable)

The katana must NEVER feel:
- mechanical
- linear
- rigid
- weightless

All motion must:
- follow continuous curved trajectories with no angular discontinuities
- express inertia through delayed response (the lerp system)
- feel like a single uninterrupted flow, not a sequence of states
- feel **inevitable**, not triggered

---

### 3-Act Scroll Story

| Act | Scroll % | Description |
|-----|----------|-------------|
| 1 | 0–30% | Katana emerges from darkness — controlled reveal, restrained inertia |
| 2 | 30–70% | Full activation — orbital motion stable, reflections intensify and flow |
| 3 | 70–100% | Motion decelerates with authority — katana settles into perfect stillness |

---

### Light Behavior

Reflections travel across the blade surface continuously. Highlights never freeze or flicker. Light intensity reacts to orientation and implied angle. Specular movement communicates curvature, sharpness, and material quality.

Light is not decoration. It is **information about form**.

---

### Performance Rules

- Preload ALL frames before the experience becomes active — no runtime interruptions
- `requestAnimationFrame` only — no `setInterval`
- webp format — optimal compression-to-fidelity ratio
- Frame loading: concurrency cap of 20 simultaneous requests — prevents connection saturation
- Mobile: skip every 2nd frame (181 frames) — preserves perception, halves bandwidth
- Cache persists for page lifetime — frames loaded once, never again

---

### PERFORMANCE REQUIREMENTS

- **First Contentful Paint:** < 1.2s (no 3D blocking render)
- **Largest Contentful Paint:** < 2.5s
- **3D Scene strategy:** Load Three.js asynchronously after page text renders. Show a static blade image (pre-rendered PNG with depth) for the first 800ms, then swap to the interactive 3D canvas once loaded. Visitor sees the blade immediately with no blank canvas flash.
- **Animation performance:** All GSAP animations use `will-change: transform, opacity` on the specific elements. Animations are hardware-accelerated (transform + opacity only — no layout-triggering properties).
- **Image optimization:** All background textures as WebP. Retina-ready (2× for texture assets).
- **Font optimization:** Preload both typefaces in `<head>`. Use font subsetting for display fonts (only the characters used on the page).
